## 二、功能模块详解

### 2.1 初始化流程（NEW!）

**触发条件**：用户首次使用或检测到未完成初始化

**流程步骤**：

| 步骤 | 名称 | 内容 | 用户操作 |
|------|------|------|---------|
| 0 | 欢迎 | 自我介绍、功能说明 | 点击「开始设置」或「稍后再说」 |
| 1 | 身材信息 | 身高、体重、体型、年龄段 | 填写表单（可选详细尺寸） |
| 2 | 风格偏好 | 喜欢的风格、不喜欢的元素 | 多选 + 开放输入 |
| 3 | 淘宝导入 | 选择导入来源（订单/购物车/收藏） | 勾选 + 时间范围 |
| 4 | 导入进度 | 自动识别服装、分类 | 查看进度、可跳过 |
| 5 | 风格报告 | AI生成风格画像 | 确认或调整 |
| 6 | 完成 | 引导日常使用 | 开始使用 |

**数据存储**：
- `user_profile` 表：身材信息、风格偏好、初始化状态
- `style_profile` 表：风格画像、洞察建议

**关键交互**：
```
用户首次打开 → 检测 init_status
├─ pending → 开始初始化流程
├─ body_info → 继续到步骤2
├─ style_info → 继续到步骤3
├─ taobao_import → 继续到步骤4
├─ style_report → 继续到步骤5
└─ completed → 跳过初始化，直接使用
```

### 2.3 搭配推荐（保留原有功能）

**功能清单**：

| 功能 | 触发方式 | 说明 |
|------|---------|------|
| 今日穿搭 | "今天穿什么？" | 结合天气、场合、衣橱推荐 |
| 场合推荐 | "明天面试穿什么？" | 场合感知智能推荐 |
| 天气适配 | "下雨天怎么穿？" | 天气感知建议 |
| 成套搭配 | "给我配一套约会穿搭" | 上衣+下装+鞋子+配饰 |
| 搭配记录 | "今天穿了XX" | 记录历史搭配 |

**推荐逻辑**（增强）：
1. **用户画像过滤**：根据身材信息推荐合身款式
2. **风格偏好匹配**：优先推荐用户喜欢的风格
3. **天气适配**：根据温度、天气调整推荐
4. **场合匹配**：通勤/约会/休闲/运动等
5. **颜色搭配**：基于色彩理论推荐配色
6. **衣橱现有**：只推荐用户已有的单品

**新增**：结合身材信息的建议
```python
# 身材建议示例
if user.body_type == "pear":
    # 梨形身材：上紧下松，突出上半身
    suggest("A字裙", "阔腿裤", "泡泡袖上衣")
    avoid("紧身裤", "包臀裙")

if user.height < 160:
    # 娇小身材：显高穿搭
    suggest("高腰裤", "短上衣", "同色系穿搭")
    avoid("超长外套", "宽松长裙")
```

## 三、数据存储设计

### 3.1 数据库表结构

```sql
-- 用户画像表（NEW!）
CREATE TABLE user_profile (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    
    -- 基本信息
    nickname TEXT,
    gender TEXT,                    -- male/female/other
    
    -- 身材信息
    height INTEGER,                 -- 身高 cm
    weight REAL,                    -- 体重 kg
    body_type TEXT,                 -- 体型：standard/pear/apple/h/inverted_triangle
    age_range TEXT,                 -- 年龄段：18-25/25-35/35-45/45+
    
    -- 详细尺寸（可选）
    shoulder_width REAL,            -- 肩宽 cm
    bust REAL,                      -- 胸围 cm
    waist REAL,                     -- 腰围 cm
    hip REAL,                       -- 臀围 cm
    leg_length REAL,                -- 腿长 cm
    
    -- 风格偏好（JSON）
    style_preferences TEXT,         -- ["极简通勤", "优雅知性"]
    avoid_elements TEXT,            -- ["露肩", "超短裙"]
    favorite_colors TEXT,            -- ["黑", "白", "米"]
    avoid_colors TEXT,               -- ["荧光色"]
    
    -- 初始化状态
    init_status TEXT DEFAULT 'pending',  -- pending/body_info/style_info/taobao_import/completed
    init_step INTEGER DEFAULT 0,
    
    created_at TEXT,
    updated_at TEXT
);

-- 单品表（扩展）
CREATE TABLE items (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    category TEXT NOT NULL,         -- outer/top/bottom/shoes/accessory
    color TEXT,
    season TEXT,                    -- 春夏/秋冬/四季
    style TEXT,                     -- JSON array
    image_path TEXT,
    
    -- 淘宝来源（NEW!）
    source TEXT DEFAULT 'manual',   -- manual/taobao_order/taobao_cart/taobao_favorite
    taobao_item_id TEXT,
    taobao_order_id TEXT,
    taobao_price REAL,
    taobao_shop TEXT,
    taobao_buy_time TEXT,
    taobao_image_url TEXT,
    taobao_product_url TEXT,
    
    created_at TEXT,
    updated_at TEXT
);

-- 搭配记录表（保留）
CREATE TABLE outfits (
    id TEXT PRIMARY KEY,
    date TEXT NOT NULL,
    occasion TEXT,
    weather TEXT,
    items TEXT,                     -- JSON array of item IDs
    rating INTEGER,
    notes TEXT,
    created_at TEXT
);

-- 种草清单表（NEW!）
CREATE TABLE wishlist (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    category TEXT,
    color TEXT,
    source TEXT,                    -- manual/taobao_cart/taobao_favorite
    taobao_item_id TEXT,
    taobao_product_url TEXT,
    taobao_image_url TEXT,
    taobao_price REAL,
    status TEXT DEFAULT 'pending',  -- pending/bought/passed
    notes TEXT,
    created_at TEXT,
    updated_at TEXT
);

-- 风格画像表（NEW!）
CREATE TABLE style_profile (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    
    -- 风格标签（AI 生成）
    style_tags TEXT,                -- JSON: ["极简通勤", "优雅知性"]
    
    -- 分析数据
    color_distribution TEXT,        -- JSON: {"黑": 30, "白": 25}
    season_distribution TEXT,       -- JSON: {"春夏": 70, "秋冬": 30}
    category_distribution TEXT,     -- JSON: {"上衣": 39, "下装": 31}
    
    -- 洞察建议
    insights TEXT,                  -- JSON array
    shopping_suggestions TEXT,      -- JSON array
    body_suggestions TEXT,          -- JSON array（基于身材）
    
    -- 用户反馈
    user_feedback TEXT,
    feedback_time TEXT,
    
    created_at TEXT,
    updated_at TEXT
);
```

## 七、关键决策点

### 7.1 初始化流程是否强制？
**决策**：可选，但强烈推荐
- 用户可以选择「稍后再说」跳过
- 后续使用时检测未初始化，友好提示
- 部分功能（身材相关推荐）需要初始化后才能使用

### 7.2 淘宝导入失败如何处理？
**决策**：优雅降级
- taobao-native 不可用 → 提示手动录入
- 导入失败 → 记录失败订单，支持重试
- 识别失败 → 标记为「待确认」，用户可手动修正

### 7.3 用户画像数据隐私？
**决策**：本地存储，不上传云端
- 所有数据存储在本地 SQLite
- 可导出/删除数据
- 淘宝订单数据仅本地使用

### 7.4 风格报告更新频率？
**决策**：增量更新 + 手动触发
- 新增单品时自动更新衣橱数据
- 风格报告需手动触发「重新生成」
- 支持查看历史报告

## 九、总结

### 优化后的 StyleBuddy v0.5.0

**核心价值升级**：
1. **一键初始化** - 从淘宝订单快速建立衣橱，冷启动成本从「手动录入几十件」降低到「点击导入」

4. **无缝联动** - 与淘宝桌面版深度整合，种草->购买一气呵成

**保留的核心能力**：
- 拍照录入衣橱
- 每日穿搭推荐
- 种草咨询建议
- 衣橱分析报告

**新增的关键能力**：
- 用户画像管理
- 淘宝数据导入
- 风格画像报告
- 身材适配推荐
